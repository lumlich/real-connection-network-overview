# Real Connection Network

**Real Connection Network (RCN)** is a digital product focused on helping people discover local communities, organize real-world events, and move from passive online interaction toward healthier, more meaningful real-life connections.

Website: https://www.realconnectionnetwork.com  
Creator / product lead: [Lumír Lichý](https://www.linkedin.com/in/lumir-lichy-166342a)

This repository is a **public product and architecture overview** of Real Connection Network.

The production source code is intentionally kept private for security, privacy and product-protection reasons.

---

## What Real Connection Network is

Real Connection Network is a quieter social platform for people who want fewer empty online interactions and more real-world community life.

The product is designed around a simple idea:

> Online tools should help people meet, build trust and do things together in real life — not keep them endlessly scrolling.

RCN helps users:

- discover local communities,
- join groups based on real interests,
- organize and attend real-world events,
- communicate in a calmer and more focused environment,
- build meaningful local relationships over time.

---

## Problem the product addresses

Many online platforms are optimized for attention, visibility and engagement metrics.

That often leads to:

- passive scrolling,
- fragmented communication,
- weak local relationships,
- too much noise,
- little connection between online interaction and real-world action.

RCN is built with a different goal: to support **local communities, real-world meetings and healthier online habits**.

---

## Product principles

Real Connection Network is designed around these principles:

- **Real-world connection first**  
  Online interaction should support real meetings, activities and communities.

- **Local relevance**  
  Communities and events should be connected to real places and nearby people.

- **Calmer communication**  
  The product avoids unnecessary noise and is designed to be less addictive than traditional social feeds.

- **Privacy and safety by design**  
  Personal data, location data and community interactions require careful handling.

- **Practical community tools**  
  The product focuses on useful features: posts, communities, events, attendance, notifications and moderation.

## High-level feature areas

RCN currently focuses on these product areas:

### Local communities

Users can discover and join local communities based on interests, location and real-world relevance.

### Community posts

Communities can share posts, updates and discussions in a focused environment.

### Real-world events

Communities can organize events connected to real places and activities.

### Attendance and check-in

Event attendance is designed around real participation, including organizer-controlled attendance confirmation.

### Notifications

The product includes notification flows for important community and event updates.

### Media and photos

The product is being designed to support image uploads in a controlled, storage-efficient and privacy-conscious way.

### Safety and privacy

The product is developed with attention to account security, data minimization, access control and careful handling of user information.

## Technical overview

The production implementation is private, but the product is built as a modern web application with a backend, frontend, database and deployment layer.

High-level technical areas include:

- Rust backend development,
- API design,
- authentication and session handling,
- PostgreSQL database,
- frontend application development,
- event and community data models,
- media storage planning,
- notification flows,
- Docker-based deployment,
- production infrastructure planning,
- security hardening,
- automated testing and iterative delivery.

The project is not only a website. It is a full product with user accounts, communities, events, permissions, notifications, media considerations and deployment architecture.

## Security and privacy approach

Because RCN works with users, communities, events and location-related information, security and privacy are central to the product.

Important principles include:

- keeping production source code private,
- avoiding public exposure of sensitive implementation details,
- minimizing personal data where possible,
- careful handling of user location information,
- access control for community and event actions,
- protection of authentication and password-reset flows,
- controlled media handling,
- production logging without unnecessary personal data,
- gradual hardening before broader public rollout.
